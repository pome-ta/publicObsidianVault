## 整理

### プロジェクトの全体


このプロジェクトは、ライブコーディングエディタであり、大きく分けると2 つの機能を持っている。

1. p5.js （やp5.sound）を実行するプレイグラウンド
2. LSP 機能を持った（codemirrorを使った）ブラウザ上でのコードエディタ

ブラウザ上のエディタを編集、書き換え処理をすることで、プレイグラウンドが、エディタのコード（sketchコード）を取得してp5 を（再）描画する。


プレイグラウンドはsandbox 化（iframe）している。main のブラウザエディタには影響を与えないようにしている。
そして、エディタとプレイグラウンドで共通させている情報は、*1つのみ* である。その情報は *sketchコード* の文字列 *のみ* である。

p5.js （p5.sound） のモジュールは、sandbox 化しているプレイグラウンドとエディタで、それぞれ *個別で* 取得をしている。
取得するモジュールのバージョンは、固定し双方でズレが発生しないようにしている。



### p5.js （やp5.sound）を実行するプレイグラウンド

p5 のsketch をglobal mode ・instance mode どちらでも描画できるようにしている。

以下にp5 を描画するsandbox 上での簡易的な概要の`index.html` を提示する。

```.html
<!doctype html>
<html lang="ja">
  <head>

    <script src="https://cdn.jsdelivr.net/npm/p5@2.3.0/lib/p5.js"></script>
    <script src="https://cdn.jsdelivr.net/npm/p5.sound@0.3.0/dist/p5.sound.js"></script>

    <script type="module">{{sketch コード流し込むところ}}</script>
    <title></title>
  </head>
  <body></body>
</html>
```

簡易な概要なので、流し込みや描画までの処理は省いてしまっているが、
注目する点としては、

```
    <script src="https://cdn.jsdelivr.net/npm/p5@2.3.0/lib/p5.js"></script>
    <script src="https://cdn.jsdelivr.net/npm/p5.sound@0.3.0/dist/p5.sound.js"></script>
```

と、モジュールを取得している部分である。

つまり、

```sketch.js
const sketch = (p) => {
  const v = 360;

  p.setup = () => {
    // put setup code here
    p.createCanvas(v, v);
    p.colorMode(p.HSL, v, 1, 1);
  };

  p.draw = () => {
    // put drawing code here
    p.background(p.frameCount % v, 1, 0.5);
  };
};

new p5(sketch);
```

（例がinstance mode であると）上記の記述でp5 が描画されることとなる。
しかし、ブラウザ上のエディタでLSP を用いた補完が欲しいため、

```.js
/**
 * @param {p5} p
 */
```

と、情報を追加し、

```sketch.js
/**
 * @param {p5} p
 */
const sketch = (p) => {
  const v = 360;

  p.setup = () => {
    // put setup code here
    p.createCanvas(v, v);
    p.colorMode(p.HSL, v, 1, 1);
  };

  p.draw = () => {
    // put drawing code here
    p.background(p.frameCount % v, 1, 0.5);
  };
};

new p5(sketch);
```

このようにしている。

instance mode にて、import 文を使って`import p5 from 'p5';` と記載するサンプルコードは公式では出ていない（過去のv1系であっても、現行のv2系であっても）。
当プロジェクトでは、v2系を強く推奨しておりv2系を使用している。

公式のサンプルコードで`import p5 from 'p5';` が無いため、当プロジェクトでも`import p5 from 'p5';` の記載がなくてもp5 が描画されるよう心がけている。

サンプルコードをそのままコピペして実行すれば、問題なく描画される。という環境が期待されるためである。


### LSP 機能を持った（codemirrorを使った）ブラウザ上でのコードエディタ

プレイグラウンド上での考え方として、

> サンプルコードをそのままコピペして実行すれば、問題なく描画される。という環境が期待されるためである。

という考え方があるので、
エディタ上にあるsketchコードの文字列も、それに倣う必要がある。
プレイグラウンドへ文字列を送る際に、正規表現を使って不要なコードを削除する。という小手先な対応は悪である。

そのような考え方があるので、エディタ上でLSP を対応させるため：

```
/**
 * @param {p5} p
 */
```

を許容し、補完ができるようにしている。

また、global mode の対応も考慮し、

[`p5.sound.js/src/app.js`](https://github.com/processing/p5.sound.js/blob/main/src/app.js) では

```.js
    const p5GlobalBridge = `
      import p5_module from 'p5';

      declare global {
        const p5: typeof p5_module;
        type p5 = p5_module;
      }
    `;
    this.createVirtualFile('file:///p5-bridge.d.ts', p5GlobalBridge);
    this.#setupATA();
    this.#ata(`import 'p5';`);
```

と、`ATA` （モジュールの取得のため）と同時に、`p5GlobalBridge` にて、補完が効くように事前処理を行なっている。
この、事前処理は現在、*あなた* がp5.sound との連携検証のために、`this.createVirtualFile('file:///p5-bridge.d.ts', p5GlobalBridge);` をコメントアウトし、`createVirtualFile` で読み込ませないようにと、指示が出ている。


## まとめ

- このプロジェクトはライブコーディングエディタである
- プレイグラウンドとブラウザエディタと2つの機能を持つ
- モジュールの取得はそれぞれ個別におこなう
- instance modeとglobal mode 双方に対応させる
- ブラウザ上のsketchコードの文字列は、当プロジェクトのみ対応の小手先コードを載せる方法を強く非難する
