エディタ上のコード:

```js
/**
 * @param {p5} p
 */
const sketch = (p) => {
  const v = 360;

  p.setup = () => {
    // put setup code here
    p.createCanvas(v, v);
    p.colorMode(p.HSL, v, 1, 1);
  };

  p.draw = () => {
    // put drawing code here
    p.background(p.frameCount % v, 1, 0.5);
  };
};

new p5(sketch);
```


## パターン1

```js
import p5_module from 'p5';

type p5 =
  p5_module & {
    __TEST_METHOD__(): void;
  };

declare global {
  const p5: typeof p5_module;
}

export {};
```

`p.` も、`p5.` も`__TEST_METHOD__`  は、出ない。
`__TEST_METHOD__` 自体も赤線。

p5.js 自体は、`p.` の以降の関数が赤線。
`new p5(sketch);` で、`p5` は赤線が無いが、`sketch` に赤線。

## パターン2

```js
      import p5_module from 'p5';
      
      declare global {
        const p5: typeof p5_module;
        type p5 = p5_module & {
          __TEST_METHOD__(): void;
        };
      }
      
      
      export {};
```

`p.` で、`__TEST_METHOD__` が出る。`p5.` では出ない。
`__TEST_METHOD__` 自体も赤線。

p5.js の`p.` に、赤線出ない。

しかし、`new p5(sketch);` で、`p5` は赤線が無いが、`sketch` に赤線。
