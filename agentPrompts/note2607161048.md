## 方針転換

```
package.json
types/
├── index.d.ts

├── core.d.ts
├── sources.d.ts
├── effects.d.ts
├── analysis.d.ts

```

この状態から、一旦0にします。

1. `package.json` と、`types/index.d.ts` のみ
2. `types/index.d.ts` で簡易な補完
3. `types/index.d.ts` で、p5.js と連携した補完
4. `types/index.d.ts` と他の`types/` の連携


といった流れを想定しています。

## 1. `package.json` と、`types/index.d.ts` のみ

名称は、`p5.sound` であるが、p5.js との連携や、p5.sound 自体のメソッド等は、一旦無しとして、ただのmodule とする。

ここを初期化状態として、一から進めて行く。

p5.js は、`ATA` にて導入済みの状態。
以降も、p5.js は`ATA` にて取得されて、エディタ上ではLSP と連携している状態。

## 2. `types/index.d.ts` で簡易な補完

とりあえず、LSP で、`p5.sound` が補完ができる状態。
p5.sound というmodule 名称ではあるが、p5.sound としての振る舞いを意識することは不要。

仮のもので、補完ができており、LSP でp5.js へ干渉していない状態を確認したい。


## 3. `types/index.d.ts` で、p5.js と連携した補完

instance mode で、`p.` や、`p5` `new p5.` としたときに、p5.sound の振る舞いとして機能するように。

ここで、過去p5.js の補完が壊れてしまった部分などを整理したい。



## 4. `types/index.d.ts` と他の`types/` の連携

`.d.ts` 間での連携や、js のソースコードを見ながら`.d.ts` を書くなど、理解を深めていきたい。
