
エディタ上では:

```
/**
 * @param {p5} p
 */
const sketch = (p) => {
  const v = 360;

  p.setup = () => {
    // put setup code here
    p.createCanvas(v, v);
    p.colorMode(p.HSL, v, 1, 1);
  };

  p.draw = () => {
    // put drawing code here
    p.background(p.frameCount % v, 1, 0.5);
  };
};

new p5(sketch);

```


この状態で、補完が出る状態を「ますは」目指している。
理由としては、LSP とは別の環境で、

```
    <script src="https://cdn.jsdelivr.net/npm/p5@2.2.3/lib/p5.js"></script>
    <script src="https://cdn.jsdelivr.net/npm/p5.sound@0.3.0/dist/p5.sound.js"></script>
```

と、`index.html` でp5.js を取得している。
なので、sketch のコードで、`import p5 from 'p5';` とimport はしないのである。





## `declare const __TEST__: P5;`


```
import type P5 from 'p5';

declare const __TEST__: P5;
```

の結果は、

```
Cannot find name '__TEST__'.
```

つまり、あなたの確認の

> 赤波線ってどっち？
> ①

と、「①」 となる。

## `declare global {const __TEST__: P5;}`


```
import type P5 from 'p5';

declare global {
  const __TEST__: P5;
}

export {};
```

で、
```
__TEST__.
```


と入力。この結果は、

`p.` の補完が出てきた、`p5.` の時の補完ではない。
